## 一、线程安全单例 🔥🔥

### 1. 双重检查锁（DCL）—— 必须默写

```java
public class Singleton {
    // volatile 必不可少：防止指令重排导致拿到"半初始化"对象
    private static volatile Singleton instance;
    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) {                 // 第一次检查：无锁，已初始化则直接返回
            synchronized (Singleton.class) {
                if (instance == null) {         // 第二次检查：拿到锁后再确认
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

> 考点：`volatile` 为什么必须有？`new` 分三步（分配内存→初始化→赋引用），可能重排成 1→3→2，别的线程在 3 之后拿到非空但未初始化的对象。`volatile` 禁止重排。

### 2. 静态内部类（推荐，懒加载 + 天然线程安全）

```java
public class Singleton {
    private Singleton() {}
    private static class Holder {               // 类加载机制保证线程安全且懒加载
        private static final Singleton INSTANCE = new Singleton();
    }
    public static Singleton getInstance() { return Holder.INSTANCE; }
}
```

> 原理：内部类在**首次被引用时才加载**，JVM 类初始化由锁保证线程安全（见 [03-JVM](./03-JVM.md) 类加载）。**枚举单例**（`enum Singleton { INSTANCE; }`）是唯一能防反射 + 防序列化破坏的写法。

## 二、生产者-消费者 / 阻塞队列 🔥🔥

### 3. 用 ReentrantLock + Condition 手写有界阻塞队列

```java
public class BoundedQueue<T> {
    private final Object[] items;
    private int head, tail, count;
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition notFull  = lock.newCondition();
    private final Condition notEmpty = lock.newCondition();

    public BoundedQueue(int cap) { items = new Object[cap]; }

    public void put(T x) throws InterruptedException {
        lock.lock();
        try {
            while (count == items.length) notFull.await();   // 满则等待(while防虚假唤醒)
            items[tail] = x;
            if (++tail == items.length) tail = 0;            // 环形
            count++;
            notEmpty.signal();                               // 唤醒消费者
        } finally { lock.unlock(); }
    }

    @SuppressWarnings("unchecked")
    public T take() throws InterruptedException {
        lock.lock();
        try {
            while (count == 0) notEmpty.await();             // 空则等待
            T x = (T) items[head];
            if (++head == items.length) head = 0;
            count--;
            notFull.signal();                                // 唤醒生产者
            return (T) x;
        } finally { lock.unlock(); }
    }
}
```

> 考点：①为什么用 `while` 不用 `if`（防虚假唤醒）；②两个 Condition 分离唤醒（避免唤醒错对象）；③`await/signal` 必须在持锁内。生产环境直接用 `ArrayBlockingQueue`/`LinkedBlockingQueue`，但手写考的是对 AQS Condition 的理解。

### 4. 三线程交替打印 ABC（Condition 版思路）

用 1 个 lock + 3 个 Condition（或一个 `state` 变量），每个线程打印后唤醒下一个，`while(state != 自己的轮次) condition.await()`。也可用 `Semaphore` 串联。

## 三、手写 LRU 缓存 🔥🔥

### 5. 继承 LinkedHashMap（最简，面试可先写这个）

```java
public class LRUCache<K, V> extends LinkedHashMap<K, V> {
    private final int capacity;
    public LRUCache(int capacity) {
        super(capacity, 0.75f, true);            // accessOrder=true：按访问顺序排序
        this.capacity = capacity;
    }
    @Override
    protected boolean removeEldestEntry(Map.Entry<K, V> eldest) {
        return size() > capacity;                // 超容量自动淘汰最久未访问
    }
}
```

### 6. 双向链表 + HashMap（LeetCode 146，考底层功底）

```java
public class LRUCache {
    private final Map<Integer, Node> map = new HashMap<>();
    private final Node head = new Node(0, 0), tail = new Node(0, 0); // 哨兵
    private final int cap;

    public LRUCache(int cap) { this.cap = cap; head.next = tail; tail.prev = head; }

    public int get(int key) {
        if (!map.containsKey(key)) return -1;
        Node n = map.get(key);
        moveToHead(n);                            // 访问即提到最前
        return n.val;
    }

    public void put(int key, int val) {
        if (map.containsKey(key)) {
            Node n = map.get(key); n.val = val; moveToHead(n); return;
        }
        if (map.size() == cap) {                  // 淘汰尾部(最久未用)
            Node last = tail.prev; remove(last); map.remove(last.key);
        }
        Node n = new Node(key, val); map.put(key, n); addToHead(n);
    }

    private void addToHead(Node n){ n.prev=head; n.next=head.next; head.next.prev=n; head.next=n; }
    private void remove(Node n){ n.prev.next=n.next; n.next.prev=n.prev; }
    private void moveToHead(Node n){ remove(n); addToHead(n); }

    static class Node { int key, val; Node prev, next; Node(int k,int v){key=k;val=v;} }
}
```

> 考点：HashMap 实现 O(1) 查找，双向链表实现 O(1) 移动/删除，哨兵节点简化边界。**LFU**（146 进阶）需再加频次桶，了解思路即可。

## 四、手写限流器 🔥（与 [24-高并发高可用](./24-高并发高可用.md) 配合）

### 7. 令牌桶（允许突发）

```java
public class TokenBucket {
    private final long capacity;        // 桶容量(突发上限)
    private final double ratePerMs;     // 每毫秒生成令牌数
    private double tokens;
    private long lastRefill = System.currentTimeMillis();

    public TokenBucket(long capacity, double ratePerSecond) {
        this.capacity = capacity; this.ratePerMs = ratePerSecond / 1000; this.tokens = capacity;
    }

    public synchronized boolean tryAcquire() {
        long now = System.currentTimeMillis();
        tokens = Math.min(capacity, tokens + (now - lastRefill) * ratePerMs); // 惰性补令牌
        lastRefill = now;
        if (tokens >= 1) { tokens -= 1; return true; }
        return false;
    }
}
```

> 关键技巧：**惰性补充**——不开后台线程定时加令牌，而是请求来时按时间差一次性补，省线程。生产直接用 Guava `RateLimiter` / Sentinel / Redis+Lua（分布式）。

### 8. 滑动窗口计数（思路）

用 `Deque<Long>` 存请求时间戳，每次请求先移除窗口外的旧时间戳，再看队列大小是否超阈值。分布式版用 Redis ZSet（score=时间戳）+ Lua 原子操作。

## 五、手写简易线程池 ⭐

```java
public class SimpleThreadPool {
    private final BlockingQueue<Runnable> queue;
    private final List<Worker> workers = new ArrayList<>();
    private volatile boolean running = true;

    public SimpleThreadPool(int poolSize, int queueCap) {
        queue = new LinkedBlockingQueue<>(queueCap);
        for (int i = 0; i < poolSize; i++) { Worker w = new Worker(); workers.add(w); w.start(); }
    }

    public void execute(Runnable task) {
        if (!running || !queue.offer(task)) throw new RejectedExecutionException(); // 拒绝策略
    }

    public void shutdown() { running = false; workers.forEach(Thread::interrupt); }

    private class Worker extends Thread {
        public void run() {
            while (running || !queue.isEmpty()) {
                try {
                    Runnable task = queue.poll(1, TimeUnit.SECONDS); // 阻塞取任务
                    if (task != null) task.run();
                } catch (InterruptedException e) { if (!running) break; }
            }
        }
    }
}
```

> 考点：worker 线程**循环从阻塞队列取任务复用**（这就是线程池"复用线程"的本质），队列满触发拒绝策略，shutdown 优雅停止。对照真实 `ThreadPoolExecutor` 七参数（见 [04-并发编程](./04-并发编程.md)）。

## 六、CompletableFuture 异步编排 🔥（实战高频）

### 9. 并行调用多个下游再聚合（接口提速利器）

```java
CompletableFuture<UserInfo>  f1 = CompletableFuture.supplyAsync(() -> userService.get(id), pool);
CompletableFuture<List<Order>> f2 = CompletableFuture.supplyAsync(() -> orderService.list(id), pool);
CompletableFuture<Integer>   f3 = CompletableFuture.supplyAsync(() -> pointService.get(id), pool);

CompletableFuture.allOf(f1, f2, f3).join();          // 等全部完成(总耗时≈最慢的一个)

Profile profile = new Profile(f1.join(), f2.join(), f3.join()); // 聚合
```

> 价值：3 个串行 300ms 的调用并行后总耗时 ≈ 100ms（见 [34-性能优化](./34-性能优化与压测.md) 反模式"串行调下游"）。
> 易错点：①**必须传自定义线程池**，否则用公共 `ForkJoinPool.commonPool` 会被打满相互拖累；②超时用 `orTimeout`（JDK 9+）；③异常用 `exceptionally`/`handle` 兜底，否则 `join` 抛 `CompletionException`。

### 10. 串行依赖与组合

- `thenApply`（同步转换）/`thenApplyAsync`（异步）；`thenCompose`（扁平化嵌套 future，类似 flatMap）；`thenCombine`（合并两个 future 结果）；`whenComplete`（完成回调，不改结果）。

## 七、其它高频小题（结论速记）

| 题 | 关键点 |
| --- | --- |

- DCL 单例为什么要 volatile？→ 防 new 的指令重排（一）。
