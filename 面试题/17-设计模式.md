# 17 · 设计模式

> 覆盖：设计原则（SOLID）、常用设计模式及其在框架中的落地。面试重点不是背 23 种，而是**能说清楚常用模式的意图 + 框架中的实例 + 手写单例**。

---

## 一、设计原则（SOLID + 其它）🔥

| 原则 | 含义 |
| --- | --- |
| 单一职责 SRP | 一个类只负责一件事 |
| 开闭原则 OCP | 对扩展开放、对修改关闭 |
| 里氏替换 LSP | 子类可替换父类且行为正确 |
| 接口隔离 ISP | 接口要小而专，不强迫实现无用方法 |
| 依赖倒置 DIP | 依赖抽象而非具体实现（面向接口编程） |
| 迪米特法则 | 最少知识，降低耦合 |
| 合成复用 | **优先组合而非继承** |

## 二、创建型

### 1. 单例模式 🔥🔥（必会手写）

**推荐写法**：

```java
// 1. 静态内部类（懒加载 + 线程安全，推荐）
public class Singleton {
    private Singleton() {}
    private static class Holder { static final Singleton I = new Singleton(); }
    public static Singleton getInstance() { return Holder.I; }
}

// 2. 双重检查锁 DCL（注意 volatile）
public class Singleton2 {
    private static volatile Singleton2 instance;
    private Singleton2() {}
    public static Singleton2 get() {
        if (instance == null) {
            synchronized (Singleton2.class) {
                if (instance == null) instance = new Singleton2();
            }
        }
        return instance;
    }
}

// 3. 枚举（最简洁，天然防反射/反序列化破坏，《Effective Java》推荐）
public enum Singleton3 { INSTANCE; }
```

- 饿汉式（静态字段直接初始化）简单但不懒加载。
- 静态内部类利用类加载机制保证线程安全 + 懒加载。
- DCL 必须 `volatile` 防止 `new` 的指令重排（见 [04-并发编程](./04-并发编程.md)）。

### 2. 工厂 / 抽象工厂 / 建造者 ⭐

- **简单工厂**：一个工厂方法按参数造对象。
- **工厂方法**：每个产品一个工厂，符合开闭原则。
- **抽象工厂**：生产一族相关产品。
- **建造者（Builder）**：分步构造复杂对象（如 `StringBuilder`、Lombok `@Builder`、`OkHttpClient.Builder`）。

框架实例：Spring `BeanFactory`、`Calendar.getInstance()`、`Executors`。

### 3. 原型模式 ⭐

通过 `clone()` 复制对象创建新实例，避免重复初始化开销（注意深浅拷贝）。

## 三、结构型

### 4. 代理模式 🔥

为目标对象提供代理以控制访问，增强功能。静态代理 / **动态代理（JDK / CGLIB）**。框架实例：**Spring AOP、MyBatis Mapper、RPC stub**。

### 5. 装饰器模式 ⭐

动态地给对象添加职责，不改变原类。框架实例：**Java IO（`BufferedInputStream` 包 `FileInputStream`）**、MyBatis `CachingExecutor`。

### 6. 适配器模式 ⭐

把一个接口转换成期望的另一个接口。实例：Spring MVC `HandlerAdapter`、`InputStreamReader`（字节转字符）。

### 7. 其它结构型 ⭐

外观（Facade，封装子系统统一入口）、桥接、组合、享元（连接池/常量池/`Integer` 缓存）。

---

## 四、行为型

### 8. 策略模式 🔥

定义一系列算法，封装并可互换，消除大量 `if-else`。实例：线程池拒绝策略、Spring `Resource` 加载、各种 `Comparator`。常配合工厂 + Map 实现“无 if-else 路由”。

### 9. 模板方法模式 🔥

父类定义算法骨架，子类实现具体步骤。实例：`JdbcTemplate`、`AbstractApplicationContext.refresh()`、MyBatis `BaseExecutor`、Servlet `HttpServlet`。

### 10. 观察者模式 🔥

对象状态变化时通知所有依赖者。实例：Spring 事件 `ApplicationListener`、各种 Listener、MQ 发布订阅。

### 11. 责任链模式 🔥

请求沿处理链传递，每个节点决定处理或传递。实例：**Servlet Filter、Spring MVC 拦截器、Netty ChannelPipeline、Sentinel/Dubbo Filter 链、网关过滤器**。

### 12. 其它行为型 ⭐

迭代器（`Iterator`）、状态、命令、备忘录、中介者、解释器、访问者。

## 五、框架中的设计模式速记 🔥

| 框架 | 模式 |
| --- | --- |
| Spring | 工厂、单例、代理、模板方法、观察者、适配器、装饰器 |
| MyBatis | 代理、工厂、建造者、装饰器、模板方法、责任链 |
| JDK | 装饰器(IO)、适配器、迭代器、享元、观察者 |
| Netty | 责任链(Pipeline)、观察者、工厂、单例 |
| Tomcat | 责任链、外观、模板方法 |
