## 一、认证与授权

### 1. 认证（Authentication）vs 授权（Authorization）🔥

- **认证**：你是谁（登录验证身份）。
- **授权**：你能做什么（权限校验）。

### 2. Session vs Token vs JWT 🔥

| | Session | Token（JWT） |
| --- | --- | --- |
| 状态 | 有状态（服务端存 session） | 无状态（信息在 token 里） |
| 存储 | 服务端内存/Redis | 客户端 |
| 扩展 | 分布式需 session 共享 | 天然适合分布式/微服务 |
| 跨域 | Cookie 跨域麻烦 | 放 Header，跨域友好 |

### 3. JWT 结构与原理 🔥

JWT = `Header.Payload.Signature`（Base64 编码，`.` 分隔）：

- **Header**：算法（如 HS256）。
- **Payload**：业务数据（用户 ID、过期时间 exp 等，**不要放敏感信息，是明文 Base64**）。
- **Signature**：对前两部分用密钥签名，防篡改。

特点与坑：

- **无状态**：服务端不存，靠签名验证，适合分布式。
- **缺点**：签发后**无法主动失效**（除非维护黑名单 / 短有效期 + refresh token）。
- 续期：access token 短期 + refresh token 长期换新。
- **注销/踢人**：配合 Redis 黑名单或版本号。

### 4. OAuth2 的四种授权模式 ⭐

OAuth2 是**授权框架**（第三方应用获取资源访问权，不暴露密码）：

- **授权码模式（authorization_code）**：最安全最常用（微信/GitHub 登录），通过临时 code 换 token。
- **简化/隐式模式（implicit）**：已不推荐。
- **密码模式（password）**：信任的自家应用。
- **客户端模式（client_credentials）**：服务间调用（无用户）。

### 5. SSO（单点登录）⭐

一处登录，多系统互通。实现：CAS、OAuth2、基于 JWT 的 token 共享。核心是认证中心统一签发凭证，各子系统校验。

### 6. RBAC 权限模型 ⭐

`用户 — 角色 — 权限`：给用户分配角色，角色绑定权限，解耦灵活。进阶：RBAC + 数据权限（行级）、ABAC（基于属性）。Spring Security / Sa-Token 是常用框架。

## 二、常见 Web 攻击 🔥

### 7. XSS（跨站脚本）

攻击者注入恶意脚本，在其他用户浏览器执行（盗 Cookie、钓鱼）。

- 类型：存储型（存数据库）、反射型（URL 参数）、DOM 型。
- **防御**：**输出转义**（HTML escape）、输入校验、CSP 内容安全策略、Cookie 设 `HttpOnly`（JS 读不到）。

### 8. CSRF（跨站请求伪造）

诱导已登录用户在不知情下发起请求（借用其 Cookie）。

- **防御**：**CSRF Token**（表单带随机 token 校验）、校验 `Referer`/`Origin`、**`SameSite` Cookie**、关键操作二次验证。

### 9. SQL 注入

拼接 SQL 导致恶意语句执行。

- **防御**：**预编译（PreparedStatement / MyBatis `#{}`）**、参数校验、最小权限账号、避免动态拼接（见 [08-MyBatis](./08-MyBatis.md)）。

### 10. 越权 🔥

- **水平越权**：访问同级别其他用户的数据（改 URL 里的 id 看别人订单）。
- **垂直越权**：低权限访问高权限功能。
- **防御**：每个请求都校验**当前用户对该资源的归属/权限**（不能只靠前端隐藏），后端强制鉴权。

### 11. 其它

DDoS（限流/高防/CDN）、重放攻击（时间戳 + nonce + 签名）、敏感信息泄露（脱敏、不返回多余字段）、文件上传漏洞（校验类型/重命名/隔离存储）。

## 三、加密与签名 🔥

### 12. 对称 vs 非对称 vs 哈希

| 类型 | 代表 | 特点 | 用途 |
| --- | --- | --- | --- |
| 对称加密 | AES、DES | 同一密钥加解密，快 | 大量数据加密 |
| 非对称加密 | RSA、ECC | 公钥加密私钥解密，慢，安全分发 | 密钥交换、签名 |
| 哈希（摘要） | SHA-256、MD5 | 不可逆、定长 | 完整性校验、密码存储 |

> HTTPS = 非对称加密**协商密钥** + 对称加密**传数据**（见 [16-计算机网络](./16-计算机网络.md)）。

### 13. 密码应该怎么存 🔥

- **绝不明文**，不用可逆加密。
- 用**加盐哈希**：`hash(password + 随机salt)`，每个用户独立 salt（防彩虹表）。
- 用**慢哈希算法**：**BCrypt / PBKDF2 / Argon2**（故意慢，抗暴力破解），不要用裸 MD5/SHA。
- Spring Security 默认 `BCryptPasswordEncoder`。

### 14. 加签验签（接口防篡改）⭐

开放接口防篡改/防重放：参数 + 时间戳 + nonce 按规则拼接，用密钥生成签名（HMAC/RSA），服务端验签 + 校验时间戳窗口 + nonce 去重。

## 高频追问清单

- JWT 怎么注销/续期？→ 短 token + refresh + Redis 黑名单（3）。
- 越权怎么防？→ 后端校验资源归属，不能只靠前端（10）。
- 密码怎么存才安全？→ BCrypt 加盐慢哈希（13）。
- XSS 和 CSRF 区别？→ 注入脚本 vs 伪造请求；转义+HttpOnly vs Token+SameSite（7/8）。
